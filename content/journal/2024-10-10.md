## Team

### Lot Metadata Design Session

Today's design session raised more questions than it answered, which is usually a sign we're getting into the right level of detail.
Key questions that came up:

How was the external products endpoint identified as the integration point? Why not use external pre-validation instead?
Is the lot metadata issue related to the bug where paid external sales were removing inventory from the newest lot instead of following FIFO order?
Could this be connected to the slow queries flagged in the system review?

We need to relate tickets to investigations more explicitly so we're not solving the same root cause from three different angles.
We also need to figure out which integration partners are using which version of our API, since the lot metadata behavior varies between versions.
